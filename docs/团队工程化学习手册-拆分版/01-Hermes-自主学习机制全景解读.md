# Hermes 自主学习机制全景解读

很多人一听到 “Hermes 会自主学习”，第一反应通常是：

它是不是会自己训练自己？  
是不是会自动变聪明？  
是不是会像人一样越聊越会？

如果按 Hermes 的实现思路来看，答案要更准确一点：

**Hermes 的“自主学习”不是神秘的自我进化，而是一套被明确设计出来的知识沉淀闭环。**

它做的事情可以概括成一句话：

**把一次次对话里的“值得长期保留的信息”和“可复用的方法”，从临时上下文里提炼出来，存进长期结构里，再在后续会话中重新用起来。**

## 1. 不是一个点，而是一整套闭环

Hermes 里的“学习”，不是某一个单独模块完成的。它更像一条完整的知识流转链路，核心大致由这几部分组成：

1. memory：保存稳定事实  
2. skill：保存可复用方法  
3. session search：跨会话回忆过去发生过什么  
4. background review：任务结束后做复盘，决定要不要沉淀知识  
5. memory provider：接外部长期记忆后端  
6. trajectory：保留交互轨迹，供后续分析或训练使用

所以如果你要真正理解 Hermes 的自主学习，一定不能把它理解成“模型自动变强”，而要理解成：

**它在工程上设计了一套从经验到记忆、从记忆到复用的机制。**

## 2. Hermes 到底在“学”什么

Hermes 学的内容，至少分成三类。

### 学“事实”

比如：

- 用户偏好什么样的回答风格
- 用户讨厌什么表达方式
- 某个项目的约定是什么
- 某个工具有什么坑
- 某类问题应该避开什么错误

这些都属于稳定事实，Hermes 会把这些内容沉淀到 memory 里。

### 学“方法”

事实是“知道什么”，方法是“知道怎么做”。

比如：

- 遇到某种错误应该怎么排查
- 某类任务的标准处理流程是什么
- 某个重复性工作怎样分步骤完成
- 哪种工作流是试过有效的

这些更像程序化经验，Hermes 把这类东西沉淀为 skill。

### 学“历史”

还有一类内容不一定直接写进 memory，也不一定立刻沉淀成 skill，它可能先作为历史会话存下来，等以后再回忆。

## 3. 为什么不是“全都记住”

Hermes 非常强调：

**不是所有信息都值得进入长期记忆。**

值得长期保存的：

- 用户偏好
- 用户纠正
- 稳定工作习惯
- 项目长期约定
- 反复出现的模式
- 经过验证的有效做法

不值得长期保存的：

- 一次性任务过程
- 临时执行状态
- 当前会话的中间步骤
- 短期 TODO
- 当下任务的流水日志

## 4. 自主学习是怎么触发的

Hermes 不是随时随地都在学习，它有明确触发机制。

### 第一层：系统提示主动引导

它会在系统提示里告诉模型：

- 稳定事实应该写 memory
- 可复用方法应该写 skill
- 临时任务状态不要乱写到 memory
- 如果某个 skill 过时了，要及时修补

### 第二层：运行时定期提醒

Hermes 还会统计：

- 距离上次 memory 写入过了多少轮
- 距离上次 skill 沉淀过了多少次工具迭代

一旦达到阈值，就会触发复盘。

## 5. 真正关键动作：后台复盘

Hermes 不想让学习动作和主任务抢注意力，所以它的思路是：

- 先把用户当前任务做好
- 结果先交付给用户
- 然后在后台再看一遍这轮对话
- 判断里面有没有值得沉淀的东西

如果有，就写入：

- memory
- 或 skill

## 6. 学到的东西分别存到哪里

- 事实类知识：存到 memory
- 方法类知识：存到 skill
- 历史过程：存到 session history，后续通过搜索拿回来

## 7. 学到之后，怎么再用起来

- memory 会在后续会话重新注入
- skill 会在合适任务里再次被调出来
- session history 可以被搜索和回忆

所以 Hermes 不是只靠 memory，而是：

**memory + skill + 历史回忆** 一起工作。

## 8. 压缩前记忆抢救为什么重要

自主学习最大的敌人之一，就是：

**信息还没来得及沉淀，就先被上下文压缩掉了。**

所以 Hermes 专门设计了“压缩前记忆抢救”，在上下文马上要被压缩前，先给模型一次最后机会，把值得长期保留的东西写进 memory。

## 9. 为什么还设计了 memory provider

因为 Hermes 不想把“记忆”只做成一个本地文件系统，它还留了外部 memory provider 这一层，让外部长期记忆系统也能接进来。

## 10. 一句话总结

**Hermes 的“自主学习”本质上不是模型自己神奇变强，而是把“值得长期保留的经验”从一次次对话里提炼出来，稳定地存下来、再稳定地用回去。**
