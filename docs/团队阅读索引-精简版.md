# 团队阅读索引-精简版

这份索引页只回答一个问题：

**不同角色应该先看什么。**

## 一、如果你是团队负责人 / 架构师

推荐顺序：

1. 《Hermes 团队工程化学习手册（拆分版）》
2. 团队学习图谱-Hermes架构流程与通讯时序
3. archive/团队学习笔记-Hermes源码学习与外部通讯整理

重点关注章节：

- 外部服务通讯机制全景解读
- 为什么适合接 Spring Boot
- 工程化价值到底对团队意味着什么
- Hermes 与 Spring Boot 交互控制架构图
- Spring Boot 接 Hermes 的标准落地手册

## 二、如果你是 Java / Spring Boot 开发

推荐顺序：

1. 项目根目录 README-快速开始.md
2. 《Hermes 团队工程化学习手册（拆分版）》
3. example-spring-client
4. example-spring-mcp-server

重点关注章节：

- Hermes 提供有哪些与外部直接交互控制的入口以及如何具体控制
- Hermes 与 Spring Boot 三种接入方式对照表
- Spring Boot 接 Hermes 的标准落地手册
